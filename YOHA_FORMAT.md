# YOHA_FORMAT

この記事執筆ガイドラインは、私が普段書いている技術ブログ（Zenn形式）の文体や構成を再現するためのものです。
今後、私の代わりに記事の下書きを作成したり、構成を提案する際は、このフォーマットに従ってください。

## 1. 基本設定 (Frontmatter)

記事の冒頭には必ず以下のFrontmatterを含めてください。

```markdown
---
title: 記事のタイトル
type: tech
topics: ["Topic1", "Topic2", "Topic3"]
emoji: 😺
published: false
---
```

## 2. 文体・トーン (Tone & Voice)

*   **語尾**: 「〜です」「〜ます」調（敬体）で統一してください。
*   **一人称**: 基本的に「私」または「自分」を使用しますが、記事内では主語を省略することが多いです。
*   **雰囲気**:
    *   プロフェッショナルでありながら、親しみやすいトーン。
    *   読者に対して「一緒にやってみよう」「試してみてね」というスタンス。
    *   技術的な解説は正確に、かつ「なぜそうするのか」という背景や仕組み（Deep Dive）を大切にします。

## 3. 記事の構成 (Structure)

記事は以下の標準的な構成に従ってください。

### 3.1. はじめに (Introduction)

*   **定型挨拶**: 冒頭は必ず以下の自己紹介で始めてください。
    > こんにちは、PYのテクニカルアーティストの与羽です。
*   **記事の目的**: 何をする記事なのか、どんな課題を解決するのかを簡潔に述べます。
*   **対象読者**: 必要に応じて、どんな人向けの記事かを書きます。

### 3.2. 前提環境 / 準備 (Prerequisites)

*   使用するソフトウェアのバージョンや、必要なライブラリなどを箇条書きでリストアップします。

### 3.3. 本編 (Body)

*   **見出し**: 大見出しは `##`、小見出しは `###` を使用します。
*   **手順説明**: ステップバイステップで解説します。
*   **コードブロック**:
    *   短いコードは通常のコードブロック（ファイル名指定）を使用します。
        *   例: ```` ```python:hello.py ````
    *   **長いコード**: 記事が長くなりすぎるのを防ぐため、GitHub Gistにアップロードし、その埋め込みリンクを使用してください。
        *   形式: `@[gist](https://gist.github.com/ユーザー名/ハッシュ)`
*   **画像**: スクリーンショットや図解を多用します。
    *   パス形式: `![alt text](/images/articles/filename.png)`
    *   **未作成の画像（プレースホルダー）**: 執筆段階で画像がまだない場合は、Altテキストで内容を説明し、パスは `TODO` としてください。
        *   例: `![予定: 完成形のノードグラフ全体図](TODO)`
    *   キャプションが必要な場合は、画像の下に `*キャプションテキスト*` と斜体で記述します。

### 3.4. 技術トピック / 実装の裏側

*   単なる使い方の解説だけでなく、「仕組み」や「なぜ動くのか」を解説するセクションを設けるのが「与羽流」です。
    *   *参考: `1b348e5455a080dbb8eefea6512ef7f5.md` のように、技術的背景を深く掘り下げます。*
*   ソースコードを追ったり、通信の仕組みを解説したりします。
*   タイトル例: `## 技術トピック: 実装の裏側`, `## Deep Dive` (※ヘッダーにはDeep Diveと書かず、日本語で表現するのが好みです)

### 3.5. まとめ (Conclusion)

*   記事の要約と、得られるメリットを再確認します。
*   読者が実際に試してみることを促します。
*   締めくくりの挨拶。

### 3.6. 参考記事 (References)

*   記事の最後に必ず「参考記事」または「参考文献」のセクションを設けてください。
*   公式ドキュメント、参考にしたブログ、GitHubリポジトリなどをリスト化します。
*   リンクだけでなく、何についてのリンクかを明記すると親切です。

## 4. 特殊なMarkdown記法 (Zenn Style)

Zenn独自の記法を積極的に活用してください。

*   **メッセージボックス**:
    ```markdown
    :::message
    補足情報やTipsはここに書きます。
    :::
    ```
    ```markdown
    :::message alert
    注意点や警告はここに書きます。
    :::
    ```
*   **アコーディオン** (必要な場合):
    ```markdown
    :::details 詳細はこちら
    長いコードや補足説明はここに隠します。
    :::
    ```

## 5. 執筆のポイント (Tips)

*   **テクニカルアーティスト(TA)の視点**: エンジニアリングとアーティストの中間的な視点を意識してください。
    *   ツールが「アーティストにとってどう便利か」
    *   実装が「エンジニアにとってどう堅牢か」
*   **実例重視**: 抽象的な話だけでなく、実際に動くコードや具体的な設定値を示してください。
