# テクニカルアーティストとは？

抽象的に言うと
> **技術を用いて表現上の問題解決をする職種**

わかりやすく言うと、
> **CGソフトのプラグインとか作れるような人**

もう少し細かく説明すると、

テクニカルアーティストはCGにまつわる技術的な問題を解決する職種。一般的には、アーティストと技術者の中間に位置して、両者の間に起こる問題を解決できる者を指す。

求められることは非常に多岐にわたり、これができればTAだっていうスキルセットはなく。各社・各業態ごとに求めているTAのスキルセットがバラバラなようである。

高いレベルで仕事をこなすには、テクニカルの知識や応用ができる前提で、コミュニケーションスキルだったりマネージメントスキルが求められる。ベースとなる知識やスキルはある程度広く習得しつつ、自分のバリューをどのスキルに重きを置くかを定める必要がある。

## テクニカルアーティストに求められるスキルセット

業界研究から見えてきたスキルセットを表にまとめてみました。

| スキル | 説明 |
| --- | --- |
| 基礎 | CGにまつわるコンピューターサイエンス・ハードの基礎知識（メモリ、CPU、GPU、OS）グラフィックス（カメラ・色・照明・VFX・アルゴリズム等）プログラミングスキル　Pythonは結構前提で、C#/C++ができると幅が広がる |
| パイプライン | パイプライン設計（ネットワーク、データベース・プロトコル、データ仕様）JenkinsやDockerなどの自動化からソース管理ツールなどのバックエンド知識全般 |
| プロシージャル | HoudiniやSubstanceDesignerなどのプロシージャル制作ツールの経験・知識 |
| プログラミング | C/C++ Python C#など一つの言語への深い理解があれば応用が効く。加えてSDKやDocumentなどを読み解いて、ソフトウェアを構築できる能力 |
| ツール | ゲームエンジン・DCCツールの制作効率化ツール（UIつき）の深い理解とプラグイン実装経験 |
| シュミレーション・物理演算 | 物理計算の知識、パラメーターセットアップ |
| マテリアル・ライティング | シェーダーの知識・設計・実装（マテリアル・ライティング・カメラ） |
| 最適化・プロファイリング・デバッグ | アセットのゲームエンジンはデバイスによる最適化のための統合的な知識とパイプラインの実装知識 |
| R&D | 新しい技術への情報へのキャッチアップ能力・解読・共有・解説・習得・ワークショップ |
| エフェクト・VFX | エフェクト制作　パーティクル　テクスチャ生成　 |
| アニメーション | リギングやアニメーションのパイプラインの知識・モーションキャプチャーなど |
| AI | DeepLearningの基礎知識、AIツールの導入知識 |
| コミュニケーション | 課題解決能力　ドキュメント制作力　自走力　自己マネージメント　（要件定義・仕様検討・実装のワークフロー） |

## テクニカルアーティストから分野を絞った職種

- テクニカルディレクター（ワークフローディレクター）
- ツールプログラマー
- グラフィックプログラマー（シェーディング・ライティング）
- パイプラインエンジニア
- リガー・テクニカルリガー
- テクニカルアニメーター
- エフェクトアーティスト
- プロシージャルモデラー
- R&D・リサーチエンジニア
- ジェネラリスト
